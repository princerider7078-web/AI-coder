    80→1.3 ASSUMPTIONS
    81→---------------
    82→1. Platform launches in Sonipat, Haryana first.
    83→2. Business owner manages inventory directly (no multi-vendor product marketplace).
    84→3. Service providers are third-party freelancers onboarded by admin.
    85→4. Payment gateway: Razorpay (India-first, supports UPI, cards, wallets).
    86→5. Delivery is managed in-house or via local courier for Phase 1.
    87→6. Mobile app: React Native (single codebase, Android + iOS).
    88→7. Backend: Node.js + Express or NestJS (developer's choice, must be RESTful).
    89→8. Database: PostgreSQL (primary) + Redis (caching/sessions).
    90→9. File storage: AWS S3 or Cloudflare R2.
    91→10. All prices in Indian Rupees (INR).
    92→11. Language: Hindi + English bilingual support (UI labels bilingual).
    93→12. Guest checkout is NOT allowed — account required for all purchases.

   107→SECTION 02 — PRODUCT VISION & SCOPE
   108→================================================================================
   109→
   110→2.1 PRODUCT OVERVIEW
   111→---------------------
   112→GROWPLANTS is a location-based eCommerce and Service Marketplace platform for
   113→the gardening ecosystem in India. It combines:
   114→
   115→  PILLAR 1: PRODUCT STORE
   116→    → Online store for plants, planters, gardening supplies.
   117→    → Owner-managed inventory.
   118→    → Delivery within serviceable pincodes.
   119→
   120→  PILLAR 2: SERVICE MARKETPLACE
   121→    → Customers can book gardening services.
   122→    → Services fulfilled by verified service providers.
   123→    → Admin approves all service providers.
   124→    → Commission-based revenue model.
   125→
   126→  PILLAR 3: ADMIN CONTROL CENTER
   127→    → Full platform management by business owner/admin.
   128→    → Order, booking, inventory, provider, and customer management.
   129→
   130→2.2 PHASE SCOPE
   131→----------------
   132→PHASE 1 (MVP — 0-6 months):
   133→  - Customer Web App + Mobile App
   134→  - Product Catalog (Plants, Planters, Gardening Products)
   135→  - Cart + Checkout + Payment
   136→  - Order Management
   137→  - Service Booking (basic)
   138→  - Service Provider Panel (basic)
   139→  - Admin Panel (full)
   140→  - Inventory Management
   141→  - Review & Rating
   142→  - Notification System (Email + Push + SMS)
   143→  - Sonipat city only
   144→
   145→PHASE 2 (6-18 months):
   146→  - Multi-city support (Haryana)
   147→  - Advanced analytics dashboard
   148→  - Featured listings for service providers
   149→  - Vendor subscription model
   150→
   151→PHASE 3 (18-36 months):
   152→  - National expansion
   153→  - AI recommendation engine
   154→  - AI plant doctor
   155→  - AR plant preview
   156→  - Advanced B2B features
   157→
   158→2.3 PLATFORM TYPES
   159→-------------------
   160→1. Customer Web App       → React.js (responsive, works on mobile browsers too)
   161→2. Customer Mobile App    → React Native (Android + iOS)
   162→3. Admin Web Panel        → React.js (desktop-first, separate app)
   163→4. Service Provider App   → React Native (shared with customer app, role-based)
   164→5. Backend API            → Node.js / NestJS (REST API)
   165→6. Database               → PostgreSQL + Redis
   166→7. File Storage           → AWS S3 / Cloudflare R2

   192→3.2 MODULE BREAKDOWN
   193→---------------------
   194→Backend modules (separate services or route groups):
   195→  - auth           → Authentication, sessions, OTP
   196→  - users          → Customer profiles, addresses
   197→  - products       → Catalog, categories, variants
   198→  - inventory      → Stock management
   199→  - cart           → Shopping cart (Redis-backed)
   200→  - orders         → Order lifecycle management
   201→  - services       → Service catalog
   202→  - bookings       → Booking lifecycle
   203→  - providers      → Service provider profiles
   204→  - payments       → Razorpay integration
   205→  - reviews        → Reviews and ratings
   206→  - notifications  → Email, SMS, push
   207→  - admin          → Admin-only endpoints
   208→  - search         → Search and filter
   209→  - locations      → City, pincode, area management
   210→  - uploads        → File/image upload to S3
   211→  - returns        → Return and refund management
   212→  - reports        → Analytics and reporting
   213→
   214→3.3 DATA FLOW: PRODUCT ORDER
   215→-----------------------------
   216→Customer → Browse Products → Add to Cart → Enter Address
   217→→ Select Delivery Slot (if applicable) → Review Order
   218→→ Payment (Razorpay) → Payment Webhook → Order Created
   219→→ Admin Notified → Admin Confirms → Delivery Assigned
   220→→ Customer Notified → Delivered → Order Completed
   221→→ Review Prompt Sent
   222→
   223→3.4 DATA FLOW: SERVICE BOOKING
   224→--------------------------------
   225→Customer → Browse Services → Select Service
   226→→ Enter Location (coordinates captured) → Select Date/Time
   227→→ Select Service Provider (or auto-assign) → Review & Confirm
   228→→ Payment (advance or COD) → Booking Created
   229→→ Provider Notified → Provider Confirms → Service Delivered
   230→→ Booking Completed → Review Prompt Sent
